# 紫禁营造志项目文件导览

本文档用于帮助后续 AI、开发者和参赛队友快速理解当前“紫禁营造志”项目结构、运行链路、核心文件职责和国赛展示状态。本文只描述当前主应用状态，不应把旧模板残留误认为当前展示功能。

## 项目概览

“紫禁营造志”是一个以紫禁城建筑细节交互为主题的单页 Web 应用，面向计算机设计大赛国赛展示。项目采用“多建筑三维识读 + 图文热点导览 + 专题可视化扩展 + 人文历史档案 + AI 辅助讲解 + 手势展陈交互”的结构。

当前页面与功能包括：

* 首页：展示项目主题、项目定位语、古典字体风格和“开启故宫建筑之旅”入口按钮。
* 地图页：使用本地故宫导览图展示建筑点位、资料来源和推荐演示路线。
* 六个建筑详情页：东北角楼、午门、太和殿、乾清宫、文华殿、武英殿均已支持 3D 识读。
* 3D 模型加载策略：详情页先显示 `3D DEPTH SAMPLE` 入口卡片，用户点击“进入 3D 识读”后才加载对应模型，避免初始页面卡顿。
* 3D 热点：使用 `@react-three/drei` 的 `Html` 将热点按钮绑定在模型空间坐标中，模型旋转时热点随视角投影移动。
* 图文内容：六个建筑仍保留图文介绍、图文热点说明、文化意义和 AI 讲解，不是只剩模型。
* 紫禁数读：展示古建筑营造、木材、屋顶、彩画等专题可视化。
* 文化拾遗：以图片 + 文字卡片方式展示宫廷建筑细节、屋脊瑞兽、故宫猫等文化补充内容，并支持手势展陈模式。
* 宫城人文志：以档案卷宗 / 宫廷事件档案式卡片展示故宫相关人文历史主题。
* 故宫时间轴：使用横向时间轴图片展示故宫重要历史节点。
* 彩画专题：作为专题知识扩展页面。
* AI 讲解：前端请求 `/api/hunyuan`，由 Express 后端代理调用腾讯混元接口。

当前主页面内容节奏为：

```text
紫禁数读
→ 文化拾遗
→ 宫城人文志
→ 故宫时间轴
```

神武门点位已经删除，当前地图不展示神武门。`placeholder` 类型逻辑可以存在，但当前没有通过神武门暴露“建设中”入口。

## 3D 热点机制

3D 热点使用模型空间坐标：

```ts
position: [x, y, z]
```

`Building3DCanvas` 通过 `@react-three/drei` 的 `Html` 将热点按钮渲染在 R3F 场景中。热点不是页面 `position: absolute` 固定按钮；模型旋转、缩放或 OrbitControls 视角变化时，热点会随 3D 场景投影移动。

点击热点后，详情页会展示：

* 当前识读对象；
* 所在位置；
* 功能作用；
* 观察提示；
* 详细说明。

已校准热点不应显示“待校准”。如果未来新增模型或新增热点，应先通过 DEV 校准工具校准坐标，再写回源码。

## 文化拾遗手势展陈模式

文化拾遗区域有“开启手势展陈”入口。点击后打开全屏沉浸式 overlay，展示 6 个文化拾遗主题。每个主题包含大图、标题、默认短文本、关键词标签和展开讲解长文本。

键盘操作：

* `ArrowLeft`：上一项；
* `ArrowRight`：下一项；
* `Enter` / `Space`：展开或收起讲解；
* `Escape`：退出展陈。

手势操作：

* 手向左划：下一项；
* 手向右划：上一项；
* 张开手掌：展开 / 收起讲解；
* 握拳：收起讲解，不直接退出 overlay。

摄像头画面仅用于浏览器本地识别，不上传视频流。手势识别只作用于“文化拾遗 · 手势展陈”overlay，不控制 3D 模型，不控制地图，不控制 AI。

## 宫城人文志

“宫城人文志”是新增的人文历史内容板块，用于补充故宫作为历史现场的人文叙事。该板块不采用图表，也不采用图片卡片，而是采用“档案卷宗 / 宫廷事件档案式”的信息呈现方式。

该板块放在“文化拾遗”六张卡片之后、故宫时间轴图片之前，形成：

```text
紫禁数读 → 文化拾遗 → 宫城人文志 → 故宫时间轴
```

### 呈现方式

视觉特点：

* 宣纸米白底；
* 宫墙红印章；
* 浅金细线；
* 墨色正文；
* 小号英文档案编号；
* 虚线分隔；
* 档案 / 奏折 / 宫廷卷宗式卡片。

功能特点：

* 默认显示短 `intro`；
* 点击“展开札记”后显示 `detail`；
* 再次点击可以收起；
* 通过 `expandedArchiveId` 控制当前展开项；
* 默认展开第一张；
* 同一时间只展开一张，保持页面整洁；
* 展开动画使用 `max-height`、`opacity`、`translateY`；
* 移动端应单列展示，不产生横向滚动。

### 当前 6 个主题

#### 1. 大朝礼仪

* 地点：太和殿 / 外朝
* 关键词：典礼、外朝、国家秩序
* 说明：太和殿不是日常办公处，而是国家礼仪被集中呈现的空间。

#### 2. 军机夜值

* 地点：军机处 / 养心殿周边
* 关键词：政务、奏折、权力中枢
* 说明：军机处从临时军务机构发展为清代处理军政大事的重要中枢。

#### 3. 正大光明

* 地点：乾清宫
* 关键词：匾额、继承、内廷政治
* 说明：一块匾额背后，连接着皇权、继承制度与内廷空间的政治象征。

#### 4. 殿试取士

* 地点：保和殿 / 科举制度
* 关键词：科举、士人、国家治理
* 说明：宫殿不仅承载礼仪，也曾成为士人进入国家治理体系的终点考场。

#### 5. 宫中日用

* 地点：内廷 / 宫廷生活
* 关键词：饮食、值役、日常运转
* 说明：紫禁城不只有大典与皇帝，也有饮食、值役、洒扫与日常运转。

#### 6. 从宫殿到博物馆

* 地点：故宫博物院
* 关键词：1925、开放、公共文化
* 说明：1925 年故宫博物院成立，昔日皇宫转化为面向公众开放的文化空间。

内容依据故宫博物院公开资料及清宫历史相关研究整理，用于补足项目的人文历史维度。页面中只保留轻量资料说明，不写成论文式引用。

## 资源路径规范

运行时代码和样式文件中不得出现本机绝对路径，例如：

```text
E:\
C:\
/Users/
Font materials
```

前端 public 资源应使用如下路径：

```text
/images/...
/models/...
/fonts/...
```

CSS 中应使用：

```css
url("/images/xxx.jpg")
```

不要写成：

```css
url("E:\\...")
```

文档中可以说明用户本地素材准备位置，但不要把本机路径写进运行时代码。如果文档中确需说明本地素材来源，应明确“仅为本地准备路径，不应进入代码”。

## 运行链路

1. `index.html` 提供根节点 `#root`。
2. `src/main.tsx` 挂载 React 应用。
3. `src/App.tsx` 控制页面状态，例如 `intro`、`map`、`detail`、`painting`。
4. 首页进入地图页。
5. 地图页点击建筑点位进入对应详情。
6. 详情页先显示 `3D DEPTH SAMPLE` 入口卡片。
7. 点击“进入 3D 识读”后挂载 `Building3DCanvas`。
8. `Building3DCanvas` 根据当前建筑 `modelPath` 加载对应模型。
9. 加载完成后显示 3D 模型和热点按钮。
10. 点击热点后显示对应说明。
11. `紫禁数读` 渲染图表可视化。
12. `文化拾遗` 渲染图片文字卡片，并可进入手势展陈 overlay。
13. `宫城人文志` 渲染档案卷宗式人文历史卡片。
14. `故宫时间轴` 渲染横向时间轴图片。
15. AI 面板向 `/api/hunyuan` 发送当前建筑和页面上下文，由 `server.cjs` 代理调用腾讯混元。
16. 图表区由 `PalaceDataCharts` 渲染。
17. 彩画专题通过页面状态进入独立专题视图。

## 本地开发方式

常用命令：

```bash
npm run dev
npm run dev:lan
npm run server
npm run start
npm run build
npm run preview
npm run lint
```

本地开发时，如果要使用 AI 讲解功能，需要同时启动两个进程：

```bash
npm run server
```

用于启动本地 Express 代理，默认监听 `3001`，提供 `/api/hunyuan`。

```bash
npm run dev
```

用于启动 Vite 前端开发服务，通常访问 `http://localhost:5173`。Vite 会把 `/api` 请求代理到 `http://127.0.0.1:3001`。

如果只运行 `npm run dev`，AI 请求会失败，并可能出现：

```text
[vite] http proxy error: /api/hunyuan
Error: connect ECONNREFUSED 127.0.0.1:3001
```

这不是 AI 逻辑坏了，而是本地后端代理没有启动。

## AI 讲解与环境变量

`src/components/AiGuidePanel.tsx` 负责输入问题、展示示例问题、提交 `/api/hunyuan` 请求、展示 AI 返回结果或错误提示。

`server.cjs` 是 Express 后端代理，负责读取服务端环境变量，调用腾讯混元兼容接口，并返回前端可展示的讲解文本。AI 密钥只应存在服务端 `.env` 或部署平台环境变量中，不应写入 `src`、`public` 或任何前端代码。本地 `.env` 不应提交到仓库。

当前 `server.cjs` 读取的环境变量包括：

```env
HUNYUAN_API_KEY=your_tokenhub_api_key_here
HUNYUAN_BASE_URL=https://tokenhub.tencentmaas.com/v1
HUNYUAN_MODEL=hunyuan-2.0-instruct-20251111
PORT=3001
HOST=0.0.0.0
CORS_ORIGIN=
```

`.env.example` 应只保留占位符，不要写真密钥。

* 推荐演示路线。

### `src/components/Building3DCanvas.tsx`

负责：

用于图片建筑或图文热点展示。主图懒加载，图片加载失败有兜底提示，不加载 3D 模型。

### `src/components/CultureGestureController.tsx`

只服务于“文化拾遗 · 手势展陈”overlay。负责打开摄像头、初始化 MediaPipe Hand Landmarker、检测手部关键点、判断左挥/右挥/张开手掌/握拳，并通过回调通知父组件；不控制地图、AI 或 3D 模型。

### `src/components/PalaceDataCharts.tsx`

渲染“紫禁数读：古建筑营造密码”图表区和彩画专题页。当前包含可视化说明；图表部分用于知识结构和建筑等级关系可视化表达，部分图表不作为精确统计数据使用。

### `src/components/AiGuidePanel.tsx`

负责 AI 问答面板，请求 `/api/hunyuan`。上下文来自当前建筑、热点和页面资料。本地开发必须依赖 `npm run server` 提供代理。

### `src/components/ErrorBoundary.tsx`

基础错误边界，避免局部组件异常导致整页白屏。

### `server.cjs`

Express 后端代理，提供 `POST /api/hunyuan`，读取服务端环境变量调用腾讯混元。存在 `dist` 时托管构建产物，本地默认与 Vite 代理配合使用。

### `vite.config.ts`

Vite 配置，包含 React 插件、别名、开发端口和 `/api` 代理。如果后端没启动会出现 `ECONNREFUSED 127.0.0.1:3001`。

## 构建产物与部署

构建：

```bash
npm run build
```

该命令执行：

```bash
tsc -b && vite build
```

Vite 会生成 `dist/` 构建产物。`dist/` 通常不需要手动提交到仓库，除非项目部署策略明确要求提交构建产物。

一体化运行：

```bash
npm run start
```

部署到支持 Node 的平台时，应设置服务端环境变量，并由 `server.cjs` 同时处理静态文件和 `/api/hunyuan`。

## 构建警告说明

当前 `npm run build` 可能仍出现以下历史警告：

1. `.env` 中 `NODE_ENV=production` 不被 Vite 推荐。
2. Tailwind `duration-[1.5s]` 歧义警告。
3. chunk 体积超过 500kB。

这些警告目前不阻断 `npm run build`。chunk 较大主要与 Three.js、ECharts、MediaPipe 等依赖有关。后续可通过懒加载、组件拆包、资源压缩继续优化；国赛前不要为了消除警告而进行大规模重构。

## Lint 状态说明

`npm run lint` 可能仍有历史问题，常见来源包括旧模板、shadcn 通用组件或 fast-refresh 规则。只要 `npm run build` 通过，主流程展示不一定受影响。后续可单独安排 lint 清理；国赛前不要为修 lint 大规模重构无关历史文件。

## 当前需要特别注意的点

1. 六个建筑都已经支持 3D 模型，但不要一次性预加载全部模型。
2. 点击“进入 3D 识读”后才加载当前建筑模型。
3. 东北角楼热点冻结，不要随意改动。
4. 五个新增建筑 3D 热点来自现有图文热点，不要恢复自己编造的热点体系。
5. DEV 热点校准工具默认隐藏，生产环境不显示。
6. `gridHelper` / `axesHelper` 调试线默认隐藏。
7. “宫城人文志”是档案卷宗式人文历史板块，不应改成图片卡片或图表。
8. 新增人文历史内容不应影响 `紫禁数读` 图表数据。
9. 不要为了丰富内容而编造未经核实的故宫历史事件。
10. 如后续扩写人文历史文案，应优先参考故宫博物院公开资料和可靠历史研究。
11. 本地 AI 需要同时启动 `npm run server` 和 `npm run dev`。
12. `.env` 不要提交。
13. 图表数据包含知识结构可视化表达，不要误写为精确统计数据。
14. 文化拾遗图片和时间轴图片应使用 `/images/culture-memory/...` 相对路径。
15. 模型文件应使用 `/models/...` 相对路径。
16. 不要在运行时代码中写入 `E:\...`、`C:\...` 等本机绝对路径。
17. 若使用第三方字体文件，需确认授权，尤其文件名中含“商用需授权”的字体。
18. 手势识别只在用户点击“开启手势识别”后请求摄像头；视频流只在浏览器本地用于识别，不上传后端。
19. 文化拾遗手势只控制展陈 overlay，不控制地图、AI 或 3D 模型。
20. `src/sections/*`、`info.md` 等旧模板残留不参与当前主应用。
21. lint 可能仍有历史问题，不应在国赛前为修 lint 大规模重构无关文件。
22. chunk 体积警告与 Three.js、ECharts、MediaPipe 等依赖有关，后续可通过懒加载/拆包优化。
